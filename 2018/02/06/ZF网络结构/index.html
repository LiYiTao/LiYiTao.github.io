ZF网络结构

发表于 2018-02-06

Faster-rcnn是基于caffe实现，caffe中的数据和参数都是以blob形式定义的，(width, height, channels, num)分别表示（宽，长，通道数，图像数）的四维数组，下文都用这个数据格式。

Input

数据输入层，这里的(1, 3, 224, 224)就表示输入1张3通道的图片，图片大小个224 x 224

im_info

对于一幅任意大小的P*Q图像，传入Fsater Rcnn前首先reshape到M*N大小，im_info=[M, N, scale_factor]则保存了此次缩放的所有信息。然后经过Conv Layers，经过4次pooling变为WxH=(M/16)x(N/16)大小，其中feature_stride=16则保存了该信息，用于后面计算anchor偏移量。

Convolution

卷积是图像处理最基本的操作，就是矩阵对应点相乘相加。

参数

  • num_output:
    输出feature map个数，就是卷积核的个数，每个输出特征图是用同一个卷积核卷积所有的输入特征图，再相加。
  • kernel_size : 卷积核的大小
  • pad : 补0长度
  • stride : 步长

    计算方法

    大小计算

    (in + 2 * pad - kernel) / stride + 1
    卷积层输入为(1, 3, 224, 224)
    根据大小计算公式，w = h = (224 + 2 x 3 - 7) / 2 + 1 = 112.5，向下取整为112。
    用了96个卷积核表示提取96种特征，该层输出为(1, 96, 112, 112)

卷积计算


Relu

Relu是一种激活函数，激活函数就好比人脑中的神经元，满足一定条件就能激活该神经元，传递信息。

计算方法

$f(x)=max(0,x)$
是对每个像素点计算，不改变大小，输出为(1, 96, 112, 112)

LRN

LRN（Local Response Normalization）局部响应归一化，对局部神经元的活动创建竞争机制，使得其中响应比较大的值变得相对更大，并抑制其他反馈较小的神经元，增强模型的泛化能力。

举个例子，假定为预测房价的例子，自变量为面积，房间数两个，因变量为房价。

那么可以得到的公式为：$y=\theta_1x_1+\theta_2x_2$

  • 当数据没有归一化的时候，面积数的范围可以从0到1000，房间数的范围一般为0到10，可以看出面积数的取值范围远大于房间数。
    损失函数：$J=(3\theta_1+600\theta_2-y)^2$

  • 当数据归一化之后，把面积范围变成和房间数相同数量级，则图像类似圆形。
    损失函数：$J=(0.5\theta_1+0.55\theta_2-y)^2$

参数

  • norm_region :
    • WITHIN_CHANNEL : 通道间的归一化，局部区域形状为local_size 1 1
    • ACROSS_CHANNELS : 通道内的归一化，局部区域块形状为1 local_size local_size
  • local_size :
    • 通道间归一化时表示求和的通道数
    • 通道内归一化时表示求和区间的边长
  • alpha : 缩放因子
  • beta : 指数项

计算方法

$f(x)=x\div(1+(\alpha/n)\sum_ix_i^2)^\beta$
n就是local_size，$x_i$是对区域内每个像素点，x是该像素点，因为是每个像素点归一化，不改变大小，输出为(1, 96, 112, 112)

Pooling

池化的本质是采样，常见的有取最值，取均值。

参数

  • kernel_size : 池化区域大小
  • stride : 步长
  • pad : 补0长度
  • pool : 池化方法，这里MAX是取最大值

计算方法

大小计算

(in + 2 * pad) / stride
池化输入是(1, 96, 112, 112)
根据大小计算公式，w = h = (112 + 2 x 1) / 2 = 57。
该层输出为(1, 96, 57, 57)

池化计算

Reshape

Reshape就是再不改变数据的情况下，重新排列数据。

参数

shape种参数对应的操作如下 :
| dim | 值意义 |
| ——- | —— |
| 0 | 保持输入大小 |
| -1 | 由其他值确定后保持总大小不变而确定 |
| 0、-1以外的具体值 | 输出为该值 |

如输入为(1, 35, 15, 15)

  • dim : 0，不变1
  • dim : 2，输出2
  • dim : -1，原来1x35x15x15 = 现在1x2x135x15，算出为135
  • dim : 0，不变15
    既输出为(1, 2, 135, 15)

Softmax

Softmax就是多个神经元的输出映射为(0, 1)间的概率，所有类的概率和为1，当某个类的预测值比较大时，该类的分流就比较大。这里是用来判断anchors是前景还是背景。

将原来输出是3,1,-3通过softmax函数一作用，就映射成为(0,1)的值，而这些值的累和为1（满足概率的性质），那么我们就可以将它理解成概率，在最后选取输出结点的时候，我们就可以选取概率最大（也就是值对应最大的）结点，作为我们的预测目标！

计算方法

$f(z^j)=\dfrac{e^{z^j}}{\sum_{i=1}^n e^{z^j}}$

Python: Proposal

proposal layer负责综合bbx回归修正和前景anchors，计算出精准的proposal，送入后续的ROIpooling层。

参数

  • module: ‘rpn.proposal_layer’，表示用rpn中的proposal_layer.py
  • layer: ‘ProposalLayer’
  • param_str: “‘feat_stride’: 16”，滑块滑动的大小

网络结构原理

计算过程

这个要先理解proposal是干什么的。

上图中展示了RPN网络的具体结构，可以看到，feature map 经过一个卷积核得到（1， 256，15，15），在RPN分成两条线

  • 上面一条通过softmax对anchors分类获得foreground和background（检测目标是foregrounnd），因为是２分类，所以它的维度是（1，18，15，15）。

    18 = 2(bg\fg) x 9(anchors)

  • 下面那条线是用于计算anchors的bounding box regression的偏移量，以获得精确的proposal。它的维度是（1，36，15，15）。

    36 = 4(坐标偏移) x 9(anchors)
    这两层的计算其实就是分别卷积18和36个1*1的kernel。

  • 而最后的proposcal层则负责综合foreground anchors和bounding box regression偏移量获取proposal，同时剔除太小和超出边界的propocals。

    训练时会随机选128个postive anchors + 128个negative anchors，所以输出为（1，256，15，15）

ROIPooling

ROIPooling是为了将不同的anchors对应的目标输出为同样的尺寸。
主要思路是将图像分割成nxn块，每一块进行最大池化，这样最后输出都是nxn。

参数

  • pooled_w = pooled_h = n（既输出的尺度）
  • spatial_scale : 表示缩放的大小（映射回原图）

计算过程

注意这里要先根据spatial_scale映射回原来的大小，再对每一块进行最大池化。
这样若输入为(1, 256, 15, 15)，输出为(1, 256, 6, 6)

Dropout

Dropout是为了防止网络过拟合，在训练时随机让网络的某些节点不工作（输出置零）。为了保证网络的每一层在训练阶段和测试阶段数据分布相同，训练完要行rescale，也就是乘以1/(1-p)。测试时就可以直接去掉dropout层。

参数

inner_product_param {
num_output: 4096
}
```

  • num_output : 输出尺度

如果输入为(1, 256, 6, 6), 输出为(1, 4096)，参数为(255x6x6)x4096。
