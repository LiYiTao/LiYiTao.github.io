1.关于PS部分可以处理什么

Regarding PS/PL partition

Q

之前要求大部分功能要再PL上实现，PS上的计算被限制。我觉得可以自己优化软硬件协同设计，只要达到目的就好。
有一些再FPGA上实现没有好处的，想放在软件上实现。

A

的确协同设计会增加性能，但比赛本身是要探索卷积神经网络在FPGA上的性能，请遵守当前的规定。
（在PS部分只能进行resize，其他预处理都不可以）

Image conversion on PS

Q

可以再PS部分把jpeg转化成RGB格式，再送到PL吗？

A

可以

FPGA PS ‘resize’

Q

是只能用resize函数吗？还是我们可以把数据分割成块，再送到PL

A

在PS可以resize和预处理，排序和转置也是允许的

Darius 卷积IP

加密的黑盒子，看不到源码，基于RTL的

术语

  • IFM——input map
  • Weights——Weight set
  • OFM——output map

    卷积参数

  • ifm-h，ifm-w——IFM的高和宽（6 ~ 32）
  • ifm-d——IFM的深度（8 ~ 512）
  • kernel-h，kernel-w——卷积核的高宽（1 ~ 16）
  • Stride——IFM的步长（0，1，2，4）
  • Pad——补0（0 ~ 16）
  • channels——通道数（8 ~ 512）
