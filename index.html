EAST场景文本检测

发表于 2018-03-20

总流程

  1. 全卷积产生文本的行级预测（可旋转的矩形或四边形）
  2. 二值化和非极大值抑制产生最终的结果

pipline流水(流程)

Multi-channel FCN

图片输入FCN，生成多通道的，像素级别的分数图(score map)和几何尺寸(geometry)。
预测通道之一是分数图，像素值在[0,1]。其余通道表示视图中围绕文字的每个像素的几何尺寸。分数代表了在同样的位置预测的几何尺寸的置信度。
我们设计了两种几何图形，并设计了不同的损失函数：

  • rotated box RBOX ：旋转矩形
  • quadrangle QUAD：四边形

    Thresholding & NMS

    阈值应用于每个区域，其中分数超过预定义阈值的几何形状被认为是有效的，并且保存为稍后的非极大值抵制。NMS之后的结果被认为是管道的最终输出。

1. 特征提取器

可以是一个使用ImageNet数据集预训练的卷积层与池化层交替的卷积神经网络，代码里改成了ResNet-50。

最后的输出层通过几个conv 1x1将特征层的32通道映射到一个Fs分数图和一个Fg多通道的几何信息图。几何信息图有 RBOX 和 QUAD 。

RBOX ：[上，右，下，左，角度]，每个像素点到四边的距离
QUAD ：8个数字来表示四个顶点到像素点的坐标位移


这样最后会生成一个分数图和几何信息图（深度为5），相当于每个点都有两个信息分数，位置。

卷积架构——设计一

  1. 卷积计算用列流水
  2. 卷积核的并行
  3. 特征图的并行

整体步骤

  1. DMA搬运input map到PPBUF（m个并行），搬运n个kernel到Coffe MEM（也是乒乓）
  2. 计算地址，PPBUF按列输入到FIFO
  3. 计算卷积，每次算完得到n个输出，直接存入DDR
  4. 搬入新的kernel（共N个）,重复2，3。
  5. 计算完一遍所有的kernel，DMA搬新的input map，重复2，3，４
  6. 直到计算完所有的M个map，再从DDR读取累加
  • M：input map的个数
  • m：每次并行的input map的个数
  • N: kernel的个数，即最后输出的个数
  • n：每次并行的kernel的个数

结构图

计算地址输入FIFO

(stride*i - pad + x)*800 + (strid*j - pad + y)

  • (i,j)：输出的位置
  • (x,y)：卷积核的大小
  • pad：补0
  • stride：步长
  • 800：输入的长度

例如第一个卷积层，输入(1,3,600,800), 输出(1,96,300,400)

  • i (0,299) , j (0,399)
  • x (0,6) , y (0,6)
  • pad = 3
  • stride = 2

设计二

按行搬入，存入FIFO，

  • 优点可以多个并行
  • 缺点kernel要重新搬

结构图

  • R：PPBUF1 + PPBUF2总大小
  • r：当前行r = i * stride
  1. 则每次取r : r + kernel_size到FIFO，计算完所有的卷积后，i+1。
  2. PPBUF1取完时，直接继续取PPBUF2，地址连续
  3. PPBUF2取完时，取r : R和0 : R - r + kernel_size

虽然可以并行，但是每次要搬运所有kernel。

Softmax

softmax是$f(z^j)=\dfrac{e^{j}}{\sum_{i=1}^n e^{i}}$，所以主要是求e的i次。
方法主要有泰勒展开，多项式拟合，分段拟合，查表法等。

  • 泰勒展开：阶数过多，需要大量的乘法器
  • 查表法：速度快只要一个时钟，但是需要很大的内存
  • 多项式拟合：同样需要较多的阶数
  • 分段拟合：比较常用，把softmax分成很多区间，再用多项式去拟合

具体实现

  1. 用matlab拟合函数，算出各个区间的权重和偏置
  2. 把权重k和偏置b存在ROM里
  3. 根据输入判断区间，取出对应的k和b
  4. 计算y = kx + b
  5. 一边求和保存到reg，一边存到BUF
  6. 除法IP核，得到结果。

AXI-LITE总线协议

发表于 2018-01-23

前言

AXI（Advanced eXtensible Interface）本是由ARM公司提出的一种总线协议，Xilinx从6系列的FPGA开始对AXI总线提供支持，此时AXI已经发展到了AXI4这个版本，所以当你用到Xilinx的软件的时候看到的都是“AIX4”的IP，如Vivado打包一个AXI IP的时候，看到的都是Create a new AXI4 peripheral。
AXI总线有三种类型：

  • AXI4：主要面向高性能地址映射通信的需求，是面向地址映射的接口，允许最大256轮的数据突发传输；

  • AXI4-Stream：面向高速流数据传输；去掉了地址项，允许无限制的数据突发传输规模，之前DMA到FIFO间数据传输就是用的AXI-Stream。

  • AXI4-Lite：是一个轻量级的地址映射单次传输接口，占用很少的逻辑单元。主要用于命令的控制。今天主要分析这个。
